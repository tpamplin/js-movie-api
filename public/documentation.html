MyFlix Documentation

Introduction

This api can be used to find information about different movies, directors, and genres.

You can:

Get all movies as a JSON object

Get data about a specific movie, director or genre as a JSON object

Add and remove users

Update user profiles

Add and remove movies from a user's list of favorites

Endpoints Overview

Request Type Request Body Response Endpoint Description
GET None. JSON /movies Returns a JSON object containing a list of all movies.
GET None. JSON /movies/:Title Returns a JSON object containing all information about a specific movie.
GET None. JSON /genres/:Name Returns a JSON object containing data about a specific genre.
GET None. JSON /directors/:Name Returns a JSON object containing information about a specific director.
POST JSON object with New User Data 201 "User Successfully Created" /users Adds a new user to the list of users.
PUT JSON object with updated data 201 "Username Changed Successfully" /users/:id/username Updates the username to the new username the user has picked.
POST JSON object with new movie data 201 "Movie added to favorites" /users/:id/favorites Adds a movie to a user's list of favorite movies.
DELETE None. 201 "Movie Removed from favorites" /users/:id/favorites/:title Removes a movie from a user's favorite list.
DELETE None. 201 "User deleted" /users/:id Removes a user from the list of users.

More Info

GET All Movies

/movies

A GET request to this url will return with a JSON object containing all informationa about all movies.

Examples

Request Body Example Response Body Example
None. [{"name": "The Princess Bride","year": "1987"},{"name": "The Matrix","year": "1999"},...
Parameters:

This request does not take any paramaters.

GET A Movie By Title

/movies/:title

A GET request to this url will return with a JSON object containing all information about a specific movie.

Examples

Request Body Example Response Body Example
None. [{"name": "The Princess Bride","year": "1987"}]
Parameters:

This request takes the title parameter. Replace ':title' in the url with the title of the movie you are looking for, replace any spaces with '%20', and the api should respond with any available information about that movie. For example, a GET request to:

/movies/Pulp%20Fiction

will return:

[{"name": "Pulp Fiction", "year": "1994"}]

GET A Genre by Name

/genres/:name

A GET request to this url will return with a JSON object containing all available information about a specific genre.

Examples

Request Body Example Response Body Example
None. [{"name": "mystery", "description": "The mystery genre is characterized by... ", "movies": ["Sherlock Holmes", ...] }]
Parameters:

This request takes the name parameter. Replace ':name' in the url with the name of the genre you are looking for, replace any spaces with '%20', and the api should respond with any available information about that genre. For example, a GET request to:

/genres/fantasy

will return:

[{"name": "fantasy", "description": "The fantasy genre is characterized by... ", "movies": ["The Neverending Story", ...]}]

POST A New User

/users

A POST request to this url will add the given user JSON object to the JSON object containing all current users.

Examples

Request Body Example Response Body Example
[{"username": "NewUser", "email": "newuser@domain.com", "favorites": []}] Status Code 201, User confirmed created successfully:
[{"username": "NewUser", "id": "..." "email": "newuser@domain.com", "favorites": []}]
Parameters:

This request does not take any paramaters.

PUT a new username

/users/:id/username

A PUT request to this url will update a specific user's username with a new username given by the user.

Examples

Request Body Example Response Body Example
["username": "NewerUser" ] Status Code 201. Username updated successfully:
[{"username": "NewerUser", "id": "..." "email": "newuser@domain.com", "favorites": [...]}]
Parameters:

This request takes the id parameter. Replace ':id' in the url with the id of the user whose username you are trying to change, the api should change the username and then send back the new updated user object. For example, a PUT request to:

/users/12/username

along with the appropriate request body, containing the new username, will update the username and return:

[{"username": "NewUserName", "id": "..." "email": "newuser@domain.com", "favorites": [...]}]

Post A Movie To A User's Favorites

/movies/:title

A POST Request to this url will add a specific movie to a specific user's favorites list.

Examples

Request Body Example Response Body Example
[{"name": "Pulp Fiction", "year": "1994"}] Status Code 201, movie added to favorites successfully.
[{"name": "Pulp Fiction", "year": "1994"}]
Parameters:

This request takes the id parameter. Replace ':id' in the url with the id of the user whose favorites list you are trying to update, the api should add the movie to the user's favorites list, and then respond with the object it just added. For example, a POST request to:

/users/12/favorites

along with the appropriate request body, containing the new movie that you wish to add to the list, would add that movie to the list and return:

[{"name": "Pulp Fiction", "year": "1994"}]

DELETE A Movie from A User's Favorites

/users/:id/favorites/:title

A DELETE request to this url will remove a movie from a specific user's favorites

Examples

Request Body Example Response Body Example
None. Status Code 201. Movie removed from favorites successfully.
Parameters:

This request takes the id parameter and the title parameter. Replace ':id' in the url with the id of the user whose favorites list you are trying to update, and ':title' with the title of the movie you wish to remove from their favorites list. Make sure to replace any spaces in the title of the movie with '%20'. The api should remove the movie from the user's favorites list, and then respond with a confirmation status code. For example, a DELETE request to:

/users/12/favorites/Pulp%20Fiction

Will remove 'Pulp Fiction' from the list and return:

Status Code 201: Movie removed from favorites successfully.

DELETE A User

/users/:id

A DELETE request to this url will delete a user from the system

Examples

Request Body Example Response Body Example
None. Status Code 201. User deleted successfully.
Parameters:

This request takes the id parameter. Replace ':id' in the url with the id of the user who you are deleting. The api should delete the user, and then respond with a confirmation status code. For example, a DELETE request to:

/users/12

Will delete the user with the id of 12 and return:

Status Code 201: User Deleted Successfully.
